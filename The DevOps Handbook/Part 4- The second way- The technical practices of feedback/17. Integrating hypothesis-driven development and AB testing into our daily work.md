### Developing for customers
When developing a feature or product, we can look at different scenarios and provide them to a user so as to make a decision on what works for them. This can be achieved through [A/B testing](https://www.oracle.com/in/cx/marketing/what-is-ab-testing/).
### Implementing A/B testing
A/B testing can be made possible by leveraging on [quick deployments to production environments]([[12. Automate and enable low risk releases]]), [feature toggle]([[12. Automate and enable low risk releases#Application-based release pattterns]]). The ability to experiment gives organizations ability to see what the customer wants/ prefers for the organization to prioritize accordingly.
### So what's next after A/B testing
After we have A/B testing in place, we can start documenting user stories as hypothesis that need to be tested in production. This provides an end-end view of the feature's life cycle and one is able to get feedback on whether the feature provides the expected value to the customer.
