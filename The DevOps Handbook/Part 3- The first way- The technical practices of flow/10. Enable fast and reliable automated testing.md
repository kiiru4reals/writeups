### Make developers work easier
Work on making developers work easier by integrating tests into version control. This is done automatically during the development stage. Some tests can be pushed as back as before commits to work as pre-commit hooks.
### Small incremental changes
Ensure that developers can push small incremental changes to the pipeline and have automated tests run from the fastest to the slowest (unit tests, acceptance tests, integration tests) before doing manual tests. To ensure tests are written have test coverage thresholds to ensure tests are written.
### Your tests should be fast
Your tests should maake it possible to catch errors as early as possible. Ensure your tests run fast and if necessary in parralel to reduce time taken.
### Test Driven and Acceptance Test Driven Development
Implement [Test Driven Development](https://www.ibm.com/think/topics/test-driven-development) and [Acceptance Test Driven Development aka Behavior Driven Development](https://cucumber.io/docs/bdd/) to ensure that developers write code that satisfies business requirements. Automate as many tests as possible to ensure higher levels of accuracy. 
### The Andon Cord
[Implement an Andon Cord]([[3. The second way- The principles of feedback#Collectively attacking problems]]) on your pipeline so as to timely resolve failures during the build stage, so that you can move code to production faster.
