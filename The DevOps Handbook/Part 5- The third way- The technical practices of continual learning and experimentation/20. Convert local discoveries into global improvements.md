### Leveraging chat rooms
Use chat rooms and bots to democratize knowledge. This makes development and operations processes transparent, easy to spot mistakes and understand how daily work is done.
### One source code repo for all your core functions/ libraries.
Build one source code repository for all your core dependencies. This makes it easy for all engineering teams to be able to understand how their services depend on each other. These libraries should also have an owner who is mandated to ensure that they are compatible with all the different consumers of that library. This makes it possible for the maintaining teams to develop knowledge on different services that their libraries are being consumed from.
### Leveraging tests
Leverage on Test Driven Development and CI/CD to automate the documentation process. Developers (especially newbies) can use outputs from tests to understand how applications work.
### Codifying Non-Functional Requirements
Codify NFRs to make it possible to accommodate fast deployability and easier maintainability of services.
### Automation of operations
Implement documentation of operations activities into the deployment pipeline by leveraging tools such as Terraform and adding operations activities to Jira. This creates knowledge that can be easily cascaded to other teams.
