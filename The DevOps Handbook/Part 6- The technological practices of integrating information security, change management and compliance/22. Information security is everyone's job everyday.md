### Always involve infosec
Involve information security teams during the beginning of the project so as to reduce situations where business is blocked at the end. Information security teams are more empowered to make better risk-based decisions and meet security objectives when involved in the beginning.
### A security based library
Implement a security-based librarywith all requirements that security has, an example is having an authentication library that can be reused by developers that has all security requirements put in place.
### Checks on the pipeline
Add checks on the pipeline to give development and operations teams quick feedback by implementing SAST, DAST, Dependency checker and code signing. This makes it quicker to build and deploy a product. 
### Mind your dependencies
Ensure that dependiencies are updated to prevent issues on the pipeline. Implement compliance checks to ensure that environments are hardened.
### Telemetry for security
Integrate telemetry for security into operations telemetry, this makes it possible for all teams to understand TTPs properly as they get to see their products get attacked in real-time.
### Secure the pipeline
Implement checks to ensure that CI/CD pipelines are not used as potential attack vectors, this can be done by regularly reviewing unit tests for malicious code, automating implementation and deployment of CI/CD tools using technologies like Terraform.
